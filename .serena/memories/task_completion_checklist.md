# Task Completion Checklist

Before considering a task complete, please ensure you have followed these steps:

1.  **Code Implementation**: The code has been written and fulfills the requirements of the task.
2.  **Testing**:
    -   Unit tests have been added or updated to cover the new functionality.
    -   All tests pass successfully. Run `make test` to verify.
3.  **Formatting**: The code has been formatted according to the project's style guide. Run `make fmt` for Python code.
4.  **Linting**: (Once a linter is configured) The code passes all linting checks.
5.  **Documentation**: Any necessary documentation changes have been made.
6.  **Dependencies**: Any new dependencies have been added to the appropriate `requirements.txt` or `package.json` file.
7.  **Database Migrations**: If the database schema has changed, a new Alembic migration has been created.
